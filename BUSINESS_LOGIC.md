# BUSINESS_LOGIC.md — Бизнес-логика системы

## Состав комплекта на 1 почтомат

| Длина (мм) | Кол-во на комплект |
|-----------|-------------------|
| 2225      | 3                 |
| 2000      | 11                |
| 1800      | 2                 |
| 1650      | 4                 |
| 1550      | 4                 |
| 1400      | 1                 |
| 1350      | 2                 |
| 1250      | 3                 |
| 1000      | 2                 |
| 780       | 2                 |
| **Итого** | **34 кабеля**     |

---

## Главные формулы

### Хватает на комплектов (по каждой длине)

```
kits_by_length[i] = floor(Остаток[i] / Кол-во_на_комплект[i])
```

### Готово комплектов (итог)

```
Готово_комплектов = MIN(kits_by_length по всем активным длинам)
```

Логика: система «упирается» в самую дефицитную позицию.  
Нельзя собрать больше комплектов, чем позволяет самая слабая длина.

---

## Узкое место

Длина, у которой `kits_by_length` минимальный.

Если несколько длин имеют одинаковый минимум — все они являются узкими местами и все получают приоритет `высокий`.

### Сколько нужно для +1 комплекта

```
remainder = Остаток % Кол-во_на_комплект

если remainder == 0:
    нужно = Кол-во_на_комплект   (т.е. целый следующий пакет)
иначе:
    нужно = Кол-во_на_комплект - remainder
```

Пример:
- Длина 2000 мм, на комплект: 11
- Остаток: 65
- `65 % 11 = 10`
- Нужно: `11 - 10 = 1 шт`
- Результат: «Сделай ещё 1 шт → будет +1 комплект»

---

## Дефицит до плана

```
Нужно_на_план = Кол-во_на_комплект × План
Дефицит = MAX(0, Нужно_на_план - Остаток)
Избыток  = MAX(0, Остаток - Нужно_на_план)
```

---

## Приоритеты позиций

| Условие | Приоритет | Цвет |
|---------|-----------|------|
| Узкое место (kits = MIN) | высокий | 🔴 красный |
| kits - MIN <= 2 | высокий | 🔴 красный |
| kits - MIN <= 5 | средний  | 🟡 жёлтый |
| kits - MIN > 5  | низкий   | 🟢 зелёный |

---

## Прогресс выполнения плана

```
Прогресс_% = MIN(100, round(Готово / План × 100))
```

## Пример расчёта

### Входные данные

| Длина | На компл. | Остаток | Хватает |
|-------|-----------|---------|---------|
| 2225  | 3         | 9       | 3       |
| 2000  | 11        | 30      | **2**   | ← МИНИМУМ = узкое место
| 1800  | 2         | 12      | 6       |
| 1650  | 4         | 20      | 5       |
| 1550  | 4         | 24      | 6       |
| 1400  | 1         | 8       | 8       |
| 1350  | 2         | 14      | 7       |
| 1250  | 3         | 18      | 6       |
| 1000  | 2         | 16      | 8       |
| 780   | 2         | 12      | 6       |

### Результат

- **Готово комплектов: 2** (MIN = 2, длина 2000 мм)
- **Узкое место: 2000 мм**
- Нужно ещё: `11 - (30 % 11) = 11 - 8 = 3 шт` для +1 комплекта
- При плане 10: осталось 8 комплектов

---

## Что происходит при внесении производства

1. Сотрудник вносит: длина 2000 мм, количество 3 шт
2. Остаток обновляется: `30 + 3 = 33`
3. `kits_by_length[2000] = floor(33 / 11) = 3`
4. Пересчитывается MIN по всем длинам
5. Новый MIN = 3 → **Готово: 3 комплекта**
6. Дельта = `3 - 2 = +1` → Показывается `+1 комплект 🎉`
