# CALCULATION_RULES.md — Правила расчёта

## Правило 1: Готово комплектов = MIN

```
Готово = MIN(floor(Остаток[i] / Кол_во_на_комплект[i]))
         для всех активных позиций i
```

Система физически не может собрать больше комплектов, чем позволяет самая дефицитная позиция.

---

## Правило 2: Узкое место

Позиция (или позиции) с минимальным значением `floor(Остаток / Кол_во)`.

Если несколько позиций имеют одинаковый минимум — **все они узкие места** и все помечаются красным.

---

## Правило 3: Нужно для +1 комплекта

```python
remainder = Остаток % Кол_во_на_комплект

если remainder == 0 и Остаток > 0:
    # Уже кратно — нужен целый следующий набор
    нужно = Кол_во_на_комплект
если remainder == 0 и Остаток == 0:
    # Ничего нет — нужен минимальный набор
    нужно = Кол_во_на_комплект
иначе:
    нужно = Кол_во_на_комплект - remainder
```

**Примеры:**

| Длина | На компл. | Остаток | Нужно для +1 |
|-------|-----------|---------|-------------|
| 2000  | 11        | 0       | 11          |
| 2000  | 11        | 5       | 6           |
| 2000  | 11        | 10      | 1           |
| 2000  | 11        | 11      | 11          |
| 2000  | 11        | 22      | 11          |
| 2000  | 11        | 30      | 3           |

## Правило 4: Приоритет позиций

Расчёт ведётся относительно текущего минимума (MIN = Готово_комплектов).

```
diff = Хватает_на_комплектов[i] - MIN

если diff == 0:              Приоритет = высокий (узкое место)
если 0 < diff <= 2:          Приоритет = высокий
если 2 < diff <= 5:          Приоритет = средний
если diff > 5:               Приоритет = низкий
```

Смысл: позиции с `diff = 1` скоро станут узким местом (когда MIN вырастет на 1).

---

## Правило 5: Дефицит до плана

```
Нужно_на_план = Кол_во_на_комплект × План_комплектов
Дефицит = MAX(0, Нужно_на_план - Остаток)
Избыток  = MAX(0, Остаток - Нужно_на_план)
```

Дефицит = сколько ещё нужно произвести для достижения плана по этой длине.

---

## Правило 6: Сортировка списка

Позиции отображаются в порядке:
1. Высокий приоритет (узкие места) — первыми
2. Средний приоритет
3. Низкий приоритет

Внутри одного уровня — по убыванию дефицита.

---

## Правило 7: Обновление остатков

При добавлении записи производства:
```
Остатки[Длина_мм] += Количество_сделано
```

Остаток **только растёт** через журнал производства.  
Уменьшение — только через ручную корректировку (функция `setStock`), которая доступна только мастеру.

---

## Правило 8: Дельта комплектов

После записи производства:
```
delta = Готово_после - Готово_до
```

Если `delta >= 1` → показать `"+N комплект(ов) 🎉"`  
Если `delta == 0` → показать `"Записано. Нужно ещё X шт для следующего"`

---

## Правило 9: Активный план

Выбор плана на день:
1. Берём записи из листа **План** с полем `Активен = TRUE`
2. Из них выбираем те, где `Дата = сегодня`
3. Если несколько — берём последнюю запись
4. Если нет на сегодня — берём последнюю активную запись вообще
5. Если нет ни одной — план = 0

---

## Правило 10: Прогресс в процентах

```
Прогресс% = MIN(100, round(Готово / План × 100))
```

При плане = 0 → прогресс = 0 (деление на ноль запрещено).  
Прогресс не может превышать 100%.

---

## Проверочные кейсы

### Кейс 1: Нулевые остатки
- Все остатки = 0
- Ожидаемый результат: Готово = 0, узкое место = первая позиция (или все)
- Все приоритеты = высокий

### Кейс 2: Одна позиция дефицитная
- Все остатки достаточны кроме одной
- Ожидаемый результат: Готово = min по этой позиции, она — узкое место

### Кейс 3: Нулевой план
- План = 0
- Ожидаемый результат: дефицит = 0 везде, прогресс = 0%, система работает

### Кейс 4: Перевыполнение плана
- Готово > План
- Ожидаемый результат: Прогресс = 100%, Осталось = 0

### Кейс 5: Кратный остаток
- Остаток = 22, на комплект = 11
- Хватает на = 2, Нужно для следующего = 11 (не 0!)
