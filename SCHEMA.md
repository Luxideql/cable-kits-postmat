# SCHEMA.md — Схема данных Google Sheets

## Лист: Спецификация

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | number | Порядковый номер |
| Длина_мм | number | Длина кабеля в мм |
| Количество_на_комплект | number | Сколько нужно в 1 комплект |
| Ячейки | string | Адрес ячейки хранения (A1:A3) |
| Группа | string | Группа (верхние/средние/нижние) |
| Активна | boolean | true — учитывается в расчёте |

**Данные по умолчанию (заполняются автоматически через `initSheets()`):**

```
1 | 2225 | 3  | A1:A3  | верхние | true
2 | 2000 | 11 | B1:B11 | средние | true
3 | 1800 | 2  | C1:C2  | средние | true
4 | 1650 | 4  | D1:D4  | средние | true
5 | 1550 | 4  | E1:E4  | средние | true
6 | 1400 | 1  | F1     | нижние  | true
7 | 1350 | 2  | G1:G2  | нижние  | true
8 | 1250 | 3  | H1:H3  | нижние  | true
9 | 1000 | 2  | I1:I2  | нижние  | true
10| 780  | 2  | J1:J2  | нижние  | true
```

## Лист: Остатки

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | number | Порядковый номер |
| Длина_мм | number | Длина кабеля |
| Остаток | number | Текущее количество на складе |
| Обновлено_время | datetime | Время последнего обновления |

**Логика обновления:**
- При каждом `addProductionLog` остаток увеличивается на `qty`
- При ручной корректировке через `setStock` — перезаписывается

## Лист: План

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | number | Порядковый номер |
| Дата | date | Дата плана (YYYY-MM-DD) |
| План_комплектов | number | Сколько комплектов нужно сделать |
| Комментарий | string | Заметка |
| Активен | boolean | Только один активный план на дату |

**Логика выбора активного плана:**
1. Ищем активный план с датой = сегодня
2. Если нет — берём последний активный
3. Если нет вообще — план = 0

## Лист: Производство

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | number | Порядковый номер |
| Дата | date | Дата записи (YYYY-MM-DD) |
| Дата_время | datetime | Точное время записи |
| Сотрудник_id | number | ID сотрудника из листа Сотрудники |
| Сотрудник | string | ФИО сотрудника |
| Telegram_id | string | Telegram ID сотрудника |
| Длина_мм | number | Длина кабеля |
| Количество_сделано | number | Сколько штук внесено |
| Комментарий | string | Заметка |

**Это исторический журнал.** Каждое внесение — новая строка.  
Остаток = сумма `Количество_сделано` за всё время (или с момента последней инвентаризации).

## Лист: Сотрудники

| Колонка | Тип | Описание |
|---------|-----|----------|
| id | number | Порядковый номер |
| Telegram_id | string | Telegram ID пользователя |
| ФИО | string | Имя сотрудника |
| Роль | string | сотрудник / мастер / руководитель / admin |
| Активен | boolean | false — заблокирован |

**Роли и доступ:**
- `сотрудник` — базовый доступ (вносить производство, видеть свой экран)
- `мастер` — + дашборд мастера, установка плана, корректировка остатков
- `руководитель` — то же, что мастер
- `admin` — то же, что мастер

**Авторегистрация:**
При первом открытии Web App, если `telegram_id` не найден в таблице,
система автоматически создаёт строку с ролью `сотрудник`.
Для повышения роли — отредактируйте таблицу вручную.

---

## Индексы и производительность

GAS не поддерживает индексы, поэтому:
- Лист Сотрудники: обычно < 100 строк → полный скан быстрый
- Лист Остатки: 10 строк (фиксировано спецификацией) → мгновенно
- Лист Производство: растёт со временем → при > 10000 строк рекомендуется архивировать старые данные в отдельный лист
