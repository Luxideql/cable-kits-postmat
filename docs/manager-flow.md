# Поток мастера / руководителя (Manager Flow)

## Вкладка «Мастер» (видна только при роли мастер/руководитель)

```
┌─────────────────────────────┐
│ ИТОГ ДНЯ                    │
│ Готово: 5  |  План: 10      │
│ Осталось: 5  |  50%         │
│ Узкое место: 2000 мм        │
└─────────────────────────────┘

┌─────────────────────────────┐
│ ⚠️ ПРОБЛЕМНЫЕ ПОЗИЦИИ        │
│ 2000 мм — дефицит: 80 шт    │
│ 2225 мм — дефицит: 21 шт    │
└─────────────────────────────┘

┌─────────────────────────────┐
│ СОТРУДНИКИ                  │
│ Иван Петров  — 45 шт, 3 пос │
│ Мария Иванова — 33 шт, 2 пос│
└─────────────────────────────┘
```

---

## Сценарий: начало смены

1. Открыть Web App
2. Вкладка **Мастер**
3. Проверить, установлен ли план
4. Если нет — установить план через GAS или в таблице

---

## Сценарий: контроль производства

```
1. В течение дня открывать вкладку «Мастер»
2. Проверять:
   - Готово комплектов → растёт?
   - Проблемные позиции → исчезают по мере производства?
   - Сотрудники → все ли внесли производство?
3. Если узкое место не меняется →
   сотрудник не вносит данные, или не производит нужную длину
```

---

## Сценарий: корректировка остатков

Если обнаружено расхождение с реальным складом:

```
Через Google Sheets:
1. Открыть лист "Остатки"
2. Найти строку с нужной длиной
3. Исправить значение в колонке "Остаток"
4. Данные обновятся при следующем запросе
```

Через API (только для менеджера):
```
POST /exec?action=setStock
Body: {
  "telegram_id": "ваш_id",
  "length_mm": 2000,
  "qty": 55
}
```

---

## Сценарий: установка нового плана

```
Через Google Sheets:
1. Открыть лист "План"
2. Добавить строку:
   id | Дата       | План_комплектов | Комментарий | Активен
   5  | 2024-01-16 | 12              | срочный     | TRUE
3. Предыдущий план на эту дату деактивируется автоматически при запросе
   (если используется API setPlan)

Через API:
POST /exec?action=setPlan
Body: {
  "telegram_id": "ваш_id",
  "plan_qty": 12,
  "date": "2024-01-16",
  "comment": "срочный заказ"
}
```

## Дашборд — что смотреть

| Показатель | Норма | Тревога |
|-----------|-------|---------|
| Прогресс к обеду | > 40% | < 25% |
| Узкое место | Меняется (прогресс идёт) | Стоит на месте |
| Проблемные позиции | Уменьшаются | Растут |
| Сотрудники | Все внесли за день | Кто-то не вносил |

---

## Добавление нового сотрудника

1. Попросить сотрудника открыть Web App (автоматическая регистрация)
2. В листе **Сотрудники** найти его строку по Telegram ID
3. Убедиться, что ФИО корректное
4. При необходимости сменить роль на `мастер`
