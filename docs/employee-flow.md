# Поток сотрудника (Employee Flow)

## Сценарий: утренний старт

```
1. Сотрудник открывает Telegram
2. Нажимает кнопку "📦 Комплекты" (Web App)
3. Экран загружается (1-2 сек)
4. Telegram.WebApp.ready() — SDK активирован
5. Авторизация по Telegram ID (автоматически)
6. Загрузка данных: getKitReadiness()
```

## Сценарий: внесение производства

### Шаг 1 — из карточки действия
```
Сотрудник нажимает "+ Внести сейчас"
→ Открывается форма с предвыбранной длиной 2000 мм
```

### Шаг 2 — через вкладку «Список»
```
Нажимает вкладку 📋
→ Видит карточки по приоритету
→ В карточке "2000 мм" нажимает "+ Внести"
→ Форма с предвыбранной длиной
```

### Шаг 3 — форма «Внести»
```
Длина: [2000 мм ▼]
        (уже выбрана)

Нужно для +1 комплекта: 3 шт

[+1]  [+5]  [+10]  [+3 (нужно)]

Количество: [3]

Комментарий: [...]

[✓ Сохранить]
```

### Шаг 4 — результат
```
Haptic: success vibration

┌─────────────────────────────┐
│ 🎉                          │
│ +1 комплект                 │
└─────────────────────────────┘

(через 0.8 сек → возврат на Итог)
```

## Сообщения системы

| Ситуация | Сообщение |
|----------|----------|
| Записано, +1 комплект | `+1 комплект 🎉` |
| Записано, без изменений | `Записано. Нужно ещё X шт` |
| Нет интернета | `Ошибка подключения. Проверьте сеть.` |
| Не выбрана длина | `Выберите длину` |
| Количество = 0 | `Введите количество > 0` |
